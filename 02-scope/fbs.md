# FBS（Function Breakdown Structure / 機能分解構造）

**作成フェーズ：要件定義**

システムが提供すべき「機能」を階層的に分解し、漏れなく抽出する。

## 1. トップレベル機能
- F1. （例）ユーザー管理
- F2. （例）タスク管理
- F3. （例）通知
- F4. （例）分析・レポート

## 2. 機能分解
### F1. ユーザー管理
- F1.1 アカウント登録
  - F1.1.1 メールアドレス認証
  - F1.1.2 プロフィール登録
- F1.2 ログイン/ログアウト
- F1.3 権限管理

### F2. タスク管理
- F2.1 タスク作成
- F2.2 タスク編集
- F2.3 タスク進捗更新
- F2.4 タスク検索・フィルタ

### F3. 通知
- F3.1 アプリ内通知
- F3.2 メール通知

### F4. 分析・レポート
- F4.1 進捗ダッシュボード
- F4.2 レポート出力

## 3. 要件とのトレーサビリティ

|--------|-----------|-----|-----|
| F1.1 | FR-01 | | |
| F2.1 | FR-02 | | |

## 注意
