# PBS（Product Breakdown Structure / 製品分解構造）

**作成フェーズ：基本設計**

[FBS](./fbs.md) で定義した機能を実現する「製品（コンポーネント）」を階層的に分解する。

## 1. システム全体構成
```
System
├── P1. フロントエンド
├── P2. バックエンドAPI
├── P3. データベース
├── P4. 外部連携
└── P5. インフラ
```

## 2. コンポーネント分解
### P1. フロントエンド
- P1.1 画面コンポーネント
  - P1.1.1 ログイン画面
  - P1.1.2 ダッシュボード画面
  - P1.1.3 タスク一覧/詳細画面
- P1.2 状態管理
- P1.3 APIクライアント

### P2. バックエンドAPI
- P2.1 認証サービス
- P2.2 タスクサービス
- P2.3 通知サービス
- P2.4 分析サービス

### P3. データベース
- P3.1 ユーザーテーブル
- P3.2 タスクテーブル
- P3.3 通知テーブル

### P4. 外部連携
- P4.1 メール送信サービス
- P4.2 認証プロバイダ（必要な場合）

### P5. インフラ
- P5.1 ホスティング環境
- P5.2 CI/CDパイプライン
- P5.3 モニタリング

## 3. FBS × PBS マトリクス
| 機能 \ 製品 | P1 フロント | P2 API | P3 DB | P4 外部 | P5 インフラ |
|------------|------------|--------|-------|---------|-------------|
| F1 ユーザー管理 | ◯ | ◯ | ◯ | ◯ | |
| F2 タスク管理 | ◯ | ◯ | ◯ | | |
| F3 通知 | ◯ | ◯ | ◯ | ◯ | |
| F4 分析 | ◯ | ◯ | ◯ | | |

## 注意
- PBSは「どう作るか（物理構造）」を示す
- WBSとの違い：PBSは成果物の分解、[WBS](./wbs.md) は作業の分解
