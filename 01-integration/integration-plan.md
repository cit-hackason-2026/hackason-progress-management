# 統合マネジメント計画

## 目的
各知識エリア（スコープ / スケジュール / コスト / 品質 / 資源 / コミュニケーション / リスク / 調達 / ステークホルダー）を横断して整合を取るための方針をまとめる。

## 変更管理プロセス
1. 変更要求をIssueとして起票
2. 影響範囲（スコープ・スケジュール・コスト・品質）を評価
3. リーダー承認のうえで関連ドキュメントを更新
4. `decision-log.md` に結果を記録

## ドキュメント管理
- すべての管理ドキュメントは本リポジトリに集約する
- 更新時はPR or コミットメッセージに変更理由を記載

## 進捗確認サイクル
- デイリー：Slack/Discordで進捗共有
- ウィークリー：全体ミーティングで計画と実績を突き合わせ
