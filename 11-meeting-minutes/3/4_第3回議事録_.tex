  \item 実装楽器の決定(フルート・クラリネット・オルガン・ドラム)
  \item 各楽器の実装方法の決定(フルート・クラリネット・オルガンの実装方法)

  \item エンターテイメント性を機能に落とし込む
  \item 貸出機材からエンターテイメント性を検討する

\end{enumerate}


%-------------------------------------------------------------------
\subsection*{議題②：設計書のレビュー結果}

\subsubsection*{決定事項}
\begin{enumerate}[label=・]
\item 評価指標の再考
\item エンターテイメント性が社会的背景と合致しているかを確認する
\item リズム楽器がパーカッションとなっているため修正
\end{enumerate}

\subsubsection*{議論内容}
社会的背景に対して、オーケストレーションシステムが解決手法として適当かどうかを再度検討した。その際に、
システムの精度を算出するための評価指標について議論した。評価指標に主観的なアンケートによる定量評価が含まれていたため、
主観評価と心理尺度法を参考にして、評価指標の算出方法を再考することとなった。
次に、エンターテイメント性が社会的背景と合致しているかを確認した。社会的背景として音楽によるリラクゼーションを提示していたため、
エンターテイメント性の方向性は刺激的な装置を避ける指針で合意した。
また、リズム楽器としてメトロノームからパーカッションに変更していたが、パーカッションは打楽器の総称であり特定の楽器を指すものではないため、
具体的なリズム楽器を決定する必要が生じた。最終的に、ドラムをリズム楽器として実装することとなった。

\subsection*{議題③：個人毎の次回までに実施する内容}

\subsubsection*{決定事項}
\begin{enumerate}[label=・]
\item 山口汐里のタスク（フルート実装の機能要件整理および試実装・同期方式の検討と実装）
\item 佐藤夏美のタスク（クラリネット実装の機能要件整理および試実装・同期方式の検討と実装）
\item 成毛海人のタスク（オルガン実装の機能要件整理・ドラムの機能リサーチ）
\item 細澤悠真のタスク（評価指標の再考・システム全体構成の実現可能性の検討・エンターテイメント機能の構成・GitHubの整備）
\item 武本龍のタスク（評価指標の再考・システム全体構成の実現可能性の検討・エンターテイメント機能の構成・GitHubの整備）
\item 酒井睦基のタスク（評価指標の再考・システム全体構成の実現可能性の検討・エンターテイメント機能の構成）

\end{enumerate}
\subsubsection*{議論内容}
今回、各楽器の基本波形と合成手法・倍音特性・ADSR設定・フィルタ設定・使用技術（Processing）を調査してもらったため、
そのリサーチ内容を基に各楽器を簡易的に実装し、リサーチ内容との差異がないかを確認するよう依頼した。
また、今回の議論を通じて、リズム楽器がドラムに正式決定したため、成毛にドラムのリサーチを依頼した。
さらに、システム全体構成を共有し方向性が確定したため、山口・佐藤に通信方式などの詳細設計を依頼した。
3年生のメンバーは、設計書のレビューで浮かび上がった問題について次回まで調査することとなった。
内容としては、評価指標の再考、システム全体構成の実現可能性の検討、エンターテイメント機能の構成である。
エンターテイメント機能については、今回2つの手法を候補として挙げ、それぞれの実現可能性を検討することとなった。
なお、エンターテイメント機能は楽曲開始のトリガーとして実装することが決定した。
1つ目の手法は、サーボモーターを用いてベビーメリーのように回転する仕組みを構築し、1周をトリガーとして楽曲を開始するものである。
2つ目の手法は、ロードセルを用いて、星を落とした際の衝撃をトリガーとする方式、または夜空の背景を広げ、その拡大に伴って衝撃を与えて楽曲を開始する方式である。
また、細澤と武本は本プロジェクト用のGitHub Organizationの作成およびIssueの整備を次回まで行うこととなった。
酒井は実装計画のWBSおよびガントチャートの再考を実施することとなった。
