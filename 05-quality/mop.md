# MOP（Measure of Performance / 性能指標）

**使用フェーズ：V字モデルの設計段階 → 単体/結合/システムテスト**

設計段階で定義される、定量的な性能基準。各テストの合否基準となる。

## MOP一覧
| ID | 性能指標 | 目標値 | 測定レベル | 対応コンポーネント | 対応MOE |
|----|---------|-------|-----------|-------------------|---------|
| MOP-01 | API応答時間（p95） | < 500ms | システムテスト | P2 API | MOE-01 |
| MOP-02 | 同時接続ユーザー数 | 50ユーザー | システムテスト | P2 / P3 | MOE-01 |
| MOP-03 | 画面初期表示時間 | < 2秒 | システムテスト | P1 フロント | MOE-01 |
| MOP-04 | 単体テストカバレッジ | 70%以上 | 単体テスト | 全体 | - |
| MOP-05 | 主要APIのエラー率 | < 1% | 結合テスト | P2 API | MOE-02 |

## テストレベルとの対応
| テストレベル | 対応MOP | 実施タイミング |
|-------------|--------|----------------|
| 単体テスト | MOP-04 | 実装時 |
| 結合テスト | MOP-05 | 機能統合時 |
| システムテスト | MOP-01〜03 | 全体結合後 |

## 測定・記録
- 各MOPは [V&V](./vandv.md) の検証（Verification）で確認する
- 測定結果は `test-results/` に日付付きで保存

## 注意
- MOPは「設計どおりの性能が出ているか」を測る
- MOPを満たしてもMOEを満たさないケースがあるため、両方の観点で検証する
