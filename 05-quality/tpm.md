# TPM（Technical Performance Measure / 技術性能指標）

**使用フェーズ：開発中の継続監視**

開発中に継続的に監視する定量指標。技術的リスクの早期発見が目的。

## MOP との違い
- MOP：テストの合否基準（点での判定）
- TPM：開発中に時系列で監視する値（線での判定）

## TPM一覧
| ID | 指標 | 目標値 | 警告閾値 | 測定頻度 | 測定方法 | 担当 |
|----|------|-------|---------|---------|---------|------|
| TPM-01 | API応答時間（平均） | < 300ms | > 400ms | デイリー | 自動計測スクリプト | |
| TPM-02 | バンドルサイズ | < 500KB | > 450KB | ビルド毎 | CI出力 | |
| TPM-03 | 単体テストカバレッジ | 70% | < 65% | PR毎 | CI出力 | |
| TPM-04 | 未解決バグ件数 | 0件（High） | High 1件以上 | デイリー | Issueトラッカー | |
| TPM-05 | CI成功率 | 95%以上 | < 90% | デイリー | CIダッシュボード | |

## アラート運用
- 警告閾値を超えた場合、デイリー会議で共有
- 2日連続で警告圏の場合は [risk-register](../08-risk/risk-register.md) へ起票
- 目標値未達でリリースに影響する場合は [decision-log](../01-integration/decision-log.md) に判断を記録
